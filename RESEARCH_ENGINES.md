# Research Engines

This document defines the S2M research engine outputs for Data Router Standalone.

## Engine 1: Product and user research

- **Goal:** identify workflows with the highest routing pain and fastest ROI.
- **Inputs:** routing failures, duplicate manual handoffs, integration friction.
- **Outputs:** top user jobs-to-be-done, priority use cases, friction-ranked backlog.

## Engine 2: Market and competitor research

- **Goal:** clarify differentiation in automation/routing markets.
- **Inputs:** iPaaS positioning, SMB automation tools, workflow orchestration alternatives.
- **Outputs:** competitor matrix, pricing pressure points, launch messaging options.

## Engine 3: Revenue model research

- **Goal:** pressure-test path to a $10M annualized 3-year trajectory.
- **Inputs:** expected conversion by tier, expansion triggers, integration-driven upsell.
- **Outputs:** plan assumptions, tier economics, growth levers by segment.

## Engine 4: Deployment and adoption research

- **Goal:** reduce activation time and increase production adoption.
- **Inputs:** onboarding friction, deploy errors, environment setup complexity.
- **Outputs:** deployment hardening actions, activation checklist, implementation templates.

## Engine 5: Security and trust research

- **Goal:** map trust blockers for enterprise and regulated buyers.
- **Inputs:** auth requirements, secrets handling, auditability expectations.
- **Outputs:** security posture priorities, trust-center doc backlog, control mapping.

## Standard output package

Each research cycle should produce:

1. Problem statement and scope
2. Findings summary
3. Prioritized recommendations
4. Asset impact (docs/UI/API changes)
5. Artifact links (commit hash, test/build results)
